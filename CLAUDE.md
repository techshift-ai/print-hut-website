# Print Hut Website

Marketing website for Print Hut, a custom printing shop in Woodbridge, Ontario.
Live at: https://print-hut-website.pages.dev

## Stack

- **Astro 5** — static site generator, islands architecture
- **Tailwind CSS 4** — CSS-first config via `@theme` in `src/styles/global.css`
- **React 19** — interactive islands (Hero, Products, Gallery, FAQ, Contact)
- **Framer Motion** — animations per the design system motion spec
- **Cloudflare Pages** — hosting with auto-deploy on merge to main

## Run Locally

```bash
npm install
npm run dev     # http://localhost:4321
npm run build   # builds to dist/
npm run preview # preview the build
```

```
    ui/

  lib/
    motion.ts            — Framer Motion variants, durations, easings
    siteData.ts          — business data (address, phone, products, pricing)
    useReducedMotion.ts  — prefers-reduced-motion hook
  pages/
    index.astro          — single-page marketing site
    llms.txt.ts          — AI crawler index (AEO)

public/
  assets/                — logo, favicon, OG image

  robots.txt             — sitemap reference
```

## Design Token Rules

All design values come from the `@theme` block in `src/styles/global.css`, which maps tokens from the Print Hut design system (`tokens.json`). Never hardcode hex colors, pixel values, or magic numbers in components.

### How tokens flow

1. `tokens.json` in the AI brain repo defines all values
2. `src/styles/global.css` maps them into Tailwind's `@theme` block
3. Components use Tailwind utilities: `bg-yellow-500`, `text-neutral-900`, `rounded-md`, etc.
4. Semantic tokens are CSS custom properties in `:root` (e.g., `var(--surface-brand)`)
5. Motion values are in `src/lib/motion.ts` — durations and easings match `tokens.json`

### Adding a new token

1. Add it to the design system's `tokens.json` first
2. Map it in the `@theme` block in `global.css`
3. Reference via Tailwind utility in components

## Tailwind 4 Specifics

- CSS-first config — no `tailwind.config.js` exists
- All customization is in `@theme` in `global.css`
- Custom font sizes use `text-display-2xl`, `text-body-md`, etc.
- Custom colors: `yellow-*`, `amber-*`, `neutral-*` scales

## Component Conventions

- **Astro components** (`.astro`) for static content: Header, Footer, Layout, BaseHead
- **React components** (`.tsx`) for interactive content: anything with animation, state, or user interaction
- **Hydration directives**: `client:load` for above-fold (Hero), `client:visible` for everything else
- **Animation pattern**: wrap in `AnimatedSection` for scroll-triggered reveal, use `StaggerContainer`/`StaggerItem` for lists
- All animations respect `prefers-reduced-motion` via the `useReducedMotion` hook

## Copy Rules

- ALL site copy comes from `clients/print-hut/copy-brief.md` in the AI brain repo
- Never use phrases from Techshift internal docs: "solve / save / grow," "outcomes over hours," etc.
- Banned words: "solutions," "premium," "leverage," "revolutionary," "one-stop shop," "passionate"
- Voice: direct, friendly, concrete. Prices and specifics over vague claims.

## SEO / AEO

- `LocalBusiness` JSON-LD in `Layout.astro`
- `FAQPage` JSON-LD in `index.astro`
- `/llms.txt` and `/llms-full.txt` for AI answer engines
- OG tags and Twitter Cards in `BaseHead.astro`
- `sitemap-index.xml` auto-generated by `@astrojs/sitemap`

## Deploy

- **Auto-deploy**: merge to `main` triggers GitHub Actions -> Cloudflare Pages
- **Manual deploy**: `npm run build && npx wrangler pages deploy dist --project-name=print-hut-website`
- **Preview deploys**: PRs get auto preview URLs
- **Branch protection**: after first production deploy, all changes go through PRs

## Quality Bar

Every change must pass all five:
1. **Aesthetic** — looks like a $2K site, not a template
2. **SEO** — structured data, meta tags, semantic HTML, sitemap
3. **AEO** — llms.txt, FAQ schema, direct answers, named entities
4. **Performance** — Lighthouse 95+, optimized images, minimal JS
5. **Accessibility** — WCAG AA, keyboard nav, focus rings, 44px touch targets, reduced motion support
